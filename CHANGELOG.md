# CHANGELOG

## v2.0.0 (2026-06-25)

从「帮人做决策」到「帮人看清自己」。代码量 214→423 行（+98%）。

### 新增：协作协议 § 🤝

整个 Skill 的开篇，不做两件事——不给人分类，不替人决定。核心是**自检镜四问**：

1. **你现在的状态？** — 清楚/有方向但说不清/只有直觉 → AI 自动切换追深浅
2. **你更需要「执行」还是「挑战」？** — 执行不质疑方向，挑战主动说代价
3. **这件事的本质？** — 完成它（紧凑）vs 想清楚它（松弛、允许推翻）
4. **「刚刚好」的尺度？** — 防bug / 性能 / 接口 / 全开 → AI 自适应质量侧重

基于回答自动调校 Shape 追问深度、Develop 质量侧重、整体节奏。同一人不同任务不同时间可以回来重答。

### 新增：愿望节 § 1.1

淘汰链之前先问「你真正想要什么体验」。人类不需要说清技术，AI 负责翻译成工程语言：

> 「我希望 AI 能更好地搜索」→ 多引擎聚合、统一返回格式、自动熔断
> 「我希望打开手机就能调」→ 高频、低延迟、本地优先

「愿望对了，形态自然对。愿望没挖出来，选什么都可能后悔。」

### 新增：淘汰链反向追问 § 1.3

之前淘汰链走到沙盒包就停了。现在加一问：**「不换形态，深耕能达到吗？」**

> 你现在已经有的框架/代码里，是不是还有没挖完的潜力？换形态是捷径还是绕路？

附带确认代价——选了沙盒包，放弃了 MCP 的标准化对外接口、Skill 的零成本修改。接受吗？

### 新增：兼容层级四层阶梯 § 1.4

淘汰链走完立刻定接口标准，四种层级选一个：

| 层级 | 调用方式 | 接口要求 |
|------|---------|---------|
| 🟢 原生 QuickJS | stdout/stderr | 纯文本即可 |
| 🟡 Package API | `{success, data, error}` | 统一包装 |
| 🟠 跨包调用 | 错误码 + retry_trace | 故障追踪链 |
| 🔴 公开 API | 语义化版本 + 向后兼容 | 完整契约 |

### 新增：按类型性能准则 § 5.2

不是泛泛的「注意性能」。按沙盒包主要做什么对号入座，四类各有精确准则和红线：

| 类型 | 核心策略 | 红线 |
|------|---------|------|
| 搜索类 | Promise.all 并发 + 去重去噪 + 熔断 | 并发 ≤ 8 |
| 文件类 | 流式读写 + 缓存 + 路径回退 | 大文件禁止整读 |
| 网络类 | 超时必设 + 重试退避 + gzip | 无超时 fetch = 定时炸弹 |
| 数据处理 | 懒加载 + 分页 + 增量计算 | 禁止全量加载到数组 |

### 新增：QuickJS 六条陷阱 § 5.3

从 Search Vault 实战踩出来的血泪清单：

1. JSON 循环引用 → 直接崩溃
2. 单次日志过多 → UI 线程阻塞
3. fetch 无超时 → 永久挂起
4. 同步循环中大量异步 → 内存堆积
5. 正则回溯爆炸 → CPU 卡死
6. 缓存设计缺失 → 每次都重新算

附带黄金法则：「在 QuickJS 里，减少操作次数 > 优化单次操作。」

### 变更：演进路线单线→双分叉 § 1.6

原来只有一条「做完之后怎么升级」。现在分两条路同时评估：

- **路径 A：当前框架深耕** — 1→1.1→1.2 稳步升级，代价小
- **路径 B：跨形态重构** — 沙盒包→MCP/Skill/拆分子包，天花板更高

附带三个评估维度：性能天花板（数据翻10倍会崩在哪）、组合可能性（会被别人引用吗）、被替代风险（平台原生支持了怎么办）。

### 新增：诊断清单自由判断模式 § 3

原来要求对照清单逐条强制检查。现在改为「地图」模式——过去踩过的坑画成地图，AI 用自己的判断力决定哪些适用、不适用、需要改造。包括：

- 13 条 Bug 预判（来自 Search Vault v4.0.0 QAQ + LOGIC）
- 6 条优化借鉴
- 依赖可行性矩阵

### 变更：自由判断边界 § 2

Explore 阶段增加了边界条款——退出码/返回值、文件路径规范、类型声明、包名/ID、用户明确说的约束——这五件事不能「差不多就行」。
